# 数据管理按钮功能分析与修改计划

## 一、按钮功能分析

### 1. reset-data (重置为默认设置)
- **功能**：将所有设置恢复到初始默认值
- **操作流程**：点击按钮 → 确认重置 → 清除所有用户设置 → 应用默认配置
- **数据处理**：清空当前设置，加载默认配置，更新localStorage
- **反馈**：显示重置成功提示

### 2. select-storage-file (选择存储文件)
- **功能**：让用户选择JSON文件作为持久化存储位置
- **操作流程**：点击按钮 → 打开文件选择对话框 → 用户选择文件 → 初始化文件存储
- **数据处理**：验证文件有效性，将设置保存到选定文件
- **反馈**：显示文件选择成功或失败提示

### 3. restore-backup (从备份恢复)
- **功能**：从用户选择的备份文件恢复设置
- **操作流程**：点击按钮 → 打开文件选择对话框 → 用户选择备份文件 → 验证文件 → 恢复设置
- **数据处理**：解析备份文件，验证数据完整性，更新当前设置
- **反馈**：显示恢复成功或失败提示

### 4. manual-backup (手动备份)
- **功能**：创建当前设置的备份文件并下载
- **操作流程**：点击按钮 → 生成备份文件 → 触发文件下载
- **数据处理**：将当前设置序列化为JSON，生成带时间戳的备份文件
- **反馈**：显示备份成功提示，自动下载备份文件

### 5. export-data (导出设置)
- **功能**：导出当前设置为JSON文件
- **操作流程**：点击按钮 → 生成设置文件 → 触发文件下载
- **数据处理**：将当前设置序列化为JSON，生成导出文件
- **反馈**：显示导出成功提示，自动下载导出文件

## 二、功能差异总结

| 按钮ID | 当前文本 | 功能差异 |
|--------|----------|----------|
| reset-data | 重置为默认设置 | 重置到初始默认值，不是恢复备份 |
| select-storage-file | 选择存储文件 | 选择持久化存储位置，不是恢复数据 |
| restore-backup | 从备份恢复 | 从备份文件恢复数据 |
| manual-backup | 手动备份 | 创建当前设置的备份文件 |
| export-data | 导出设置 | 导出当前设置为文件 |
