# 修复URL输入框验证问题

## 1. 问题分析
HTML输入框的type属性设置为了`url`，这会导致浏览器进行原生的URL验证。对于没有协议前缀的URL（如"www.baidu.com"），浏览器的原生URL验证会失败，从而显示"请输入网址"的提示，而不是使用我们自定义的URL验证逻辑。

## 2. 修复方案
将HTML输入框的type属性从`url`改为`text`，这样就可以完全使用我们自定义的`validateAndNormalizeUrl`方法进行URL验证和规范化，而不是依赖浏览器的原生URL验证。

## 3. 代码修改

### 3.1 修改HTML输入框
将以下代码：
```html
<input type="url" id="edit-url" required placeholder="网址">
```

替换为：
```html
<input type="text" id="edit-url" required placeholder="网址">
```

同样修改编辑现有导航项的输入框：
```html
<input type="text" id="edit-existing-url" required placeholder="网址">
```

## 4. 预期效果
- 修复后，浏览器不会再对URL进行原生验证，而是使用我们自定义的验证逻辑
- "www.baidu.com"等无协议URL不会再触发浏览器的原生验证错误
- 完全使用我们的`validateAndNormalizeUrl`方法进行验证和规范化
- 保持对所有URL格式的正确处理
