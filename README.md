## 项目概述

### 项目定位

浏览器导航主页是一款轻量级、纯前端的个性化导航工具，专注于提供简洁、高效的网站访问体验。它允许用户自由定制导航网格、壁纸背景和布局设置，同时支持多种数据存储方式，确保数据的安全性和可移植性。

### 目标用户

- 希望提高上网效率的普通用户
- 喜欢个性化定制的浏览器用户
- 需要分类管理大量常用网站的专业人士
- 寻求替代浏览器默认主页的用户

### 核心价值

- **高度可定制**：从导航项到壁纸，从布局到颜色，几乎所有元素都可以自定义
- **数据安全**：支持多种本地存储方式，确保用户数据不会泄露
- **跨浏览器兼容**：适配所有现代浏览器，提供一致的用户体验
- **轻量级**：纯前端实现，无外部依赖，加载速度快
- **易用性**：直观的操作界面，简单易用的设置选项

### 技术亮点

- 采用适配器设计模式实现多种存储方案的灵活切换
- 模块化的代码结构，便于扩展和维护
- 响应式设计，适配不同屏幕尺寸
- 事件驱动的交互设计，提供流畅的用户体验
- 分层存储策略，根据浏览器支持情况自动选择最佳存储方案

## 核心功能

### 🌐 基本功能

- **自定义导航网格**：可自由添加、编辑、删除和排序网站导航图标
- **多种搜索引擎支持**：内置Google、百度、Bing、Yahoo等多种搜索引擎，支持本地搜索
- **壁纸设置**：支持上传自定义壁纸，支持恢复默认壁纸
- **布局调整**：可调整列数、间距、图标大小和搜索框透明度
- **数据管理**：支持设置的导入/导出和重置

### 🎯 高级功能

- **标签筛选功能**：为导航项添加标签，支持按标签筛选和查看
- **工具组功能**：将多个导航项组织成工具组，方便分类管理
- **右键菜单**：支持导航图标的右键操作（编辑、删除、移动、添加到工具组）
- **拖拽排序**：支持导航图标的拖拽排序
- **快捷方式文字颜色设置**：可自定义快捷方式文字颜色，解决壁纸与文字对比度问题
- **快捷方式图标在线网址输入**：支持直接输入在线图标URL，无需本地文件上传
- **响应式设计**：适配不同屏幕尺寸，提供一致的用户体验

### 💾 数据存储

- **分层存储策略**：根据浏览器支持情况自动选择最佳存储方案
  - **File System Access API**（优先，Chrome/Edge/Opera支持）
  - **IndexedDB**（备选，所有现代浏览器支持）
  - **localStorage**（后备，所有浏览器支持）
- **本地文件存储**：支持将设置保存到本地JSON文件，提供额外的数据安全性和备份选项
- **多浏览器兼容存储**：针对Firefox等浏览器提供特殊的存储适配器，确保功能一致性

### 🎨 外观定制

- **自定义文字颜色**：可调整快捷方式文字颜色，确保在不同壁纸上都清晰可见
- **搜索框透明度调整**：根据个人喜好调整搜索框的透明度
- **灵活的网格布局**：支持调整列数（6-10列）、间距（5-30px）和图标大小（32-80px）

## 快速开始

### 1. 环境要求

- 现代浏览器（Chrome 60+、Firefox 55+、Safari 12+、Edge 79+）
- 本地HTTP服务器（可选，用于本地开发）

### 2. 安装方法

#### 方法一：直接使用

1. 下载项目文件到本地
2. 解压文件到任意目录
3. 使用浏览器打开 `index.html` 文件即可使用

#### 方法二：本地服务器（推荐，用于完整功能体验）

使用本地HTTP服务器可以获得更好的功能支持，特别是本地文件存储功能。

##### Windows系统

```bash
# 使用Python 3
python -m http.server 8000

# 或使用Node.js（需要先安装Node.js）
npx http-server -p 8000

# 或使用PowerShell内置HTTP服务器（PowerShell 7+）
Start-FileServer -Port 8000 -Root .
```

##### macOS系统

```bash
# 使用Python 3
python3 -m http.server 8000

# 或使用Node.js
npx http-server -p 8000

# 或使用Ruby
ruby -run -ehttpd . -p8000
```

##### Linux系统

```bash
# 使用Python 3
python3 -m http.server 8000

# 或使用Node.js
npx http-server -p 8000

# 或使用PHP（需要安装PHP）
php -S localhost:8000
```

### 3. 访问应用

在浏览器中访问：`http://localhost:8000`

### 4. 设置为浏览器主页

将导航页设置为浏览器主页，以便每次打开浏览器时自动访问：

#### Chrome/Edge

1. 点击浏览器右上角的三个点菜单
2. 选择"设置" > "外观"
3. 在"主页"部分，选择"自定义"，输入 `http://localhost:8000`

#### Firefox

1. 点击浏览器右上角的三条横线菜单
2. 选择"设置" > "主页和新标签页"
3. 在"主页"部分，选择"自定义URL"，输入 `http://localhost:8000`

#### Safari

1. 点击屏幕顶部的"Safari"菜单
2. 选择"偏好设置" > "通用"
3. 在"主页"字段中，输入 `http://localhost:8000`

### 5. 开始使用

- 点击"设置"按钮打开设置面板
- 自定义壁纸、布局和导航项
- 享受个性化的导航体验

### 6. 基本操作流程

1. **添加导航项**：点击"添加快捷方式"按钮或在设置面板中添加
2. **编辑导航项**：右键点击导航图标，选择"编辑网址"
3. **调整布局**：在设置面板中调整列数、间距和图标大小
4. **设置壁纸**：在设置面板中上传自定义壁纸
5. **管理数据**：在设置面板中导入/导出或重置设置

## 常见问题解答

### Q: 点击"启用本地文件存储"后提示"您的浏览器不支持文件系统访问API"？

A: 这是由于浏览器兼容性问题导致的。系统检测到您的浏览器不支持`File System Access API`，但您仍然可以使用其他存储方案。

**解决方案**：
1. **继续使用当前浏览器**：系统会自动选择最佳的替代存储方案（IndexedDB或localStorage），不影响核心功能
2. **无需更换浏览器**：系统已优化为支持所有现代浏览器，包括Firefox最新版
3. **使用导入/导出功能**：无论使用哪种浏览器，都可以通过"导出设置"和"导入设置"功能手动备份和恢复数据

### Q: 如何备份我的设置？

A: 有两种方法备份设置：
1. **使用导入/导出功能**：在设置面板的"数据管理"部分，点击"导出设置"下载JSON文件
2. **使用本地文件存储**：如果浏览器支持，可以启用本地文件存储，设置会自动保存到本地JSON文件

### Q: 如何恢复默认设置？

A: 在设置面板的"数据管理"部分，点击"重置为默认设置"按钮，确认后即可恢复默认设置。

### Q: 如何调整导航项的顺序？

A: 有两种方法调整导航项顺序：
1. **拖拽排序**：直接拖拽导航图标到目标位置
2. **右键菜单**：右键点击导航图标，选择"上移"或"下移"

## 核心类与API

### StorageAdapter（存储适配器基类）

**功能**：定义存储适配器的统一接口，实现不同存储方案的抽象

**主要方法**：
- `async openStorage()`：打开存储，读取数据
- `async saveStorage()`：保存数据到存储
- `async backupStorage()`：备份存储数据
- `async restoreStorage()`：从备份恢复数据

### FileSystemAccessAdapter（Chrome/Edge/Opera存储适配器）

**功能**：基于File System Access API实现的存储适配器，提供最佳的本地文件存储体验

**主要方法**：
- `async openStorage()`：打开本地JSON文件，读取设置
- `async saveStorage()`：保存设置到本地JSON文件
- `async backupStorage()`：创建设置的备份文件
- `async restoreStorage()`：从备份文件恢复设置

### FirefoxStorageAdapter（Firefox存储适配器）

**功能**：专为Firefox浏览器设计的存储适配器，使用替代方案实现本地文件存储

**主要方法**：
- `async openStorage()`：通过文件选择器打开设置文件

### 网址验证与规范化

**功能**：验证并规范化用户输入的网址，确保其格式正确

**实现原理**：
1. **输入预处理**：去除网址首尾空格
2. **协议自动补充**：对缺少协议的网址自动添加`https://`协议前缀
3. **格式验证**：使用`URL`构造函数验证网址格式的有效性
4. **主机名验证**：确保网址包含有效的主机名
5. **规范化输出**：返回符合标准URL格式的规范化网址

**支持的网址格式**：

| 网址格式 | 示例 | 处理结果 |
|---------|------|---------|
| http://开头 | http://www.example.com | http://www.example.com/ |
| https://开头 | https://www.example.com | https://www.example.com/ |
| www.开头 | www.example.com | https://www.example.com/ |
| 无协议 | example.com | https://example.com/ |
| 带路径 | example.com/path | https://example.com/path |
| 带查询参数 | example.com?query=value | https://example.com/?query=value |
| 带锚点 | example.com#anchor | https://example.com/#anchor |

**规范化规则**：

1. **协议补充**：
   - 对于以`www.`开头的网址，自动添加`https://`协议前缀
   - 对于其他没有协议的网址，自动添加`https://`协议前缀
   - 保留原始协议（http://或https://）

2. **格式规范化**：
   - 确保网址符合标准URL格式
   - 自动补充缺失的斜杠
   - 规范化查询参数和锚点

3. **错误处理**：
   - 空网址：提示"网址不能为空"
   - 无效格式：提示具体错误信息
   - 无效主机名：提示"网址格式无效，请检查主机名"

**常见无效网址案例及解决方案**：

| 无效网址 | 错误原因 | 解决方案 |
|---------|---------|---------|
| `example` | 缺少主机名后缀 | 添加完整域名：example.com |
| `http://` | 缺少主机名 | 添加完整主机名：http://example.com |
| `http://.com` | 无效主机名 | 添加有效的主机名：http://example.com |
| `example..com` | 无效域名格式 | 修正域名格式：example.com |
| `http:// example.com` | 包含空格 | 移除空格：http://example.com |
